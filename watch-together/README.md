# 放映厅 · 私密同步观影(房主模式)

支持 2-4 人:**创建房间的人自动成为房主**,由房主选视频源、控制播放/暂停/进度,其他人跟着看 + 开麦语音聊天 + 文字留言板。手机、电脑浏览器都能用。

## 房主机制说明

- 谁先创建房间(进入时房间码留空),谁就是房主
- 只有房主能看到"选视频源"的操作区,也只有房主的播放器有控制条(播放/暂停/拖进度)
- 其他人的播放器是只读的,画面会跟着房主自动同步
- 如果房主中途离开,系统会自动把房间里下一位提升为新房主,聊天区会有提示

## 支持的视频源

1. **YouTube 链接** —— 用官方 API 精确同步进度,最稳
2. **直接视频文件链接**(`.mp4` / `.webm` 等结尾的直链)—— 用原生 `<video>` 标签播放,同样精确同步,不依赖对方网站配合
3. **上传本地视频文件** —— 房主直接把视频文件传到服务器,大家都从你的服务器看,进度精确同步,是最"万能"的方式,不挑视频来源

**做不到的事(如实说明):** 像 B站、腾讯视频这种普通网页播放链接,浏览器技术上无法嵌入并程序控制播放进度(这是平台限制,不开放这个接口),所以这个方案里没有支持,建议改用上面第2、3种方式。

## 本地运行

```bash
cd watch-together
npm install
npm start
```

浏览器打开 `http://localhost:3000`,开两个窗口可以自己模拟房主+观众的效果。

## 让朋友也能加入(重要)

`localhost` 只有你自己电脑能访问。要让朋友加入,需要部署到一个朋友也能访问、且是 **HTTPS** 的地址(浏览器的麦克风权限 `getUserMedia` 只在 `localhost` 或 HTTPS 下可用)。

1. **免费云平台部署**(推荐):传到 GitHub,用 [Render](https://render.com)、[Railway](https://railway.app) 或 [Fly.io](https://fly.io) 部署,它们支持 Node.js 长连接服务并自带 HTTPS 域名。
2. **内网穿透临时测试**:本地跑起来后用 `ngrok http 3000` 生成临时公网 HTTPS 地址。

部署后,进入房间时地址栏会变成 `https://你的域名/#room=房间码`,把这个链接发给朋友即可。

## 关于视频上传的存储

上传的视频文件存放在服务器的 `uploads/房间码/` 目录下,当前限制单个文件最大 500MB(可以在 `server.js` 里的 `multer` 配置调整)。这是最简单直接的本地磁盘存储,没做过期清理和持久化保证,长期使用建议自己定期清理 `uploads` 目录或换成云存储。

## 关于语音通话

语音走 WebRTC 点对点连接(P2P mesh),2-4 人的小房间够用。如果个别人的网络环境特殊(公司网络、严格 NAT),P2P 直连可能建立不起来,需要加 TURN 中转服务器(比如 [Twilio](https://www.twilio.com/stun-turn) 或自建 coturn),当前只配了 Google 免费 STUN,家庭/移动网络下基本没问题。

## 文件说明

- `server.js`:房间管理(含房主机制)、播放状态转发(校验发送者是否房主)、视频上传接口、语音信令转发
- `public/index.html` / `style.css` / `client.js`:前端页面与逻辑,视频源支持 YouTube / 直链 / 上传三种
